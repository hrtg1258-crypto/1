项目操作指南（简明版）

1. 项目里有哪些关键文件

1) 数据源
- docs/data.js：上线用的数据（GitHub Pages 从 /docs 发布）
- composer/data.js：本地开发用的数据（如果你平时打开 composer/index.html）

2) 测试页（做题、出结果）
- docs/index.html：上线入口页面
- docs/app.js：测试逻辑（出题、计分、结果展示）

3) 数据看板（全量数据列表 + 搜索）
- docs/list.html：看板页面
- docs/list.js：看板逻辑

4) 样式与图片
- docs/style.css：样式
- docs/image/*：本地图片资源（例如 glg.webp、xbg.jpg）


2. 题库怎么维护（推荐方式：按作曲家分组维护）

现在题库用的是“可维护结构”：
- data.questionBank：按作曲家分组写题（你主要改这里）
- data.questions：自动从 questionBank 生成（你一般不需要手动改）

2.1 在哪里加题
打开 docs/data.js，找到 data.questionBank，例如：

  \"贝多芬\": [
      \"你说话直接，不喜欢拐弯抹角？\",
      \"遇到不爽会直接表达，不爱忍气吞声？\"
  ],

你要新增题目，只需要在数组里多加一行字符串：
  \"贝多芬\": [
      \"...旧题\",
      \"...旧题\",
      \"新增：你更愿意用行动解决问题，而不是反复讨论？\"
  ],

保存后刷新页面，就会自动出现在测试里。

2.2 每题的 5 个选项文案如何单独设置（积分不变）
默认情况下，每题用 data.defaultChoices 的 5 个文案，顺序固定对应分值：
10 / 6 / 3 / 1 / 0

如果你想给某一题做“更有趣的选项文案”，把该题从字符串改成对象：

  \"勋伯格\": [
      {
          title: \"你敢于打破规则，但也尊重传统根基？\",
          choices: [\"我天生就爱拆规则\",\"偶尔想打破一下\",\"无所谓\",\"更喜欢稳妥\",\"完全不想\"]
      },
      \"你喜欢创新，同时又重视逻辑与结构？\"
  ]

注意：
- choices 必须按顺序写 5 个，对应分值 10/6/3/1/0
- 不写 choices 的题，会自动使用 defaultChoices

2.3 题目顺序怎么控制（可选）
默认出题顺序：按 data.composers 里作曲家的顺序逐个拼接题目。

如果你想手动控制“作曲家出场顺序”，可以加一个数组 data.quizOrder：

  data.quizOrder = [\"贝多芬\",\"莫扎特\", ...]

不写 quizOrder 就按默认顺序。

2.4 题号 id 要不要维护
不需要。系统会自动从 1 开始给 questions 生成 id。


3. 作曲家库怎么改

作曲家在 data.composers 里，一条就是一个作曲家：
- name：作曲家名（题库用它做 key，尽量不要随意改）
- tag：用中文顿号“、”分隔多个标签
- image：头像路径
  - 网络图：\"https://...\"
  - 本地图：\"image/xxx.jpg\"（实际对应 docs/image/xxx.jpg）
- desc：结果页文案
- stats：六维雷达图，长度必须为 6（1~10 分）


4. 计分规则（不变）

每题都有 5 个选项，对应分值固定：
- 10 / 6 / 3 / 1 / 0

每道题都“指向”一个作曲家（to），该题得分会累加到这个作曲家总分上。
最后总分最高者为结果；并列时按稳定规则决胜（保证同样答题得到同样结果）。


5. 结果页额外信息

5.1 Top 3 候选
结果页会额外展示 Top 3，总分接近时可以看到差距。

5.2 相性（相性最好/最差）
结果页会展示“相性最好的作曲家”和“相性最差的作曲家”。
相性算法基于六维 stats 的相似度（越像越相性好，差异越大越相性差）。


6. GitHub Pages 发布（用 docs 目录）

把代码 push 到 GitHub 后，在仓库 Settings -> Pages：
- Branch 选 main
- Folder 选 /docs

保存后，Pages 会发布 docs 目录里的网站。
